Polkadot JS Network Explorer

When you open Polkadot JS apps website, you are presented with this network explorer interface! As you can see, there is a lot of information to be deciphered here, especially if you are a new user. Also, keep in mind that this interface might get updated when new features get added to Polkadot ecosystem. Alright, this tutorial is being recorded when Polkadotâ€™s runtime version is 9080. Polkadot is a blockchain. Right now, polkadot has over 6 million blocks in the chain which are produced at an interval of roughly 6 seconds. The interface provides you a snapshot of the block number, block hash and the validator who validated the block. This activity reassures us that the blockchain is alive and active.

Now, If you observe the validators closely, you would see that some of them have known and verified identities and some of them do not. We shall cover the topic of on chain indentity in a different video, but know that Polkadot supports that feature. The most recent block number corresponds to the best block number that you see here. A block has to be finalized to become part of the polkadot chain. Block finalisation takes time and hence, the finalized block number always trails behind the best block number. If block finalisation runs into issues, you would see a larger disparity between these two numbers. Here, you can see the snapshot the types of events that were captured in the respective blocks

Here is the Total number of DOTs that were issued. Iâ€™ll discuss about epoch shortly when I talk about staking. If you are a blockchain enthusiast, you have an option to explore the contents of pretty much any block on the chain, observe forks on the chain in real time and check out some information about the full node that is providing this UI for you.

Now, if you are someone interested in the stakeholders of the network, navigate to the staking page under the network dropdown. This page shows detailed view of validator and nominator statistics. Here, you can see that the network supports a maximum of 297 active validators and currently it is maxed out. In fact, there are over 617 validators right now that are waiting to become an active validator. Now, the number of nominators supported by polkadot runtime is 22500. Just be aware that these numbers are subject to change through on-chain governance.

Now, although there might be 22500 nominators on-chain, it is not guaranteed that every nominator is rewarded. If the nominator ends up backing a set of validators and none of them make it to the active validator set, the nominator will not earn any rewards in this era (which is one day on Polkadot). The mechanics of nominating and validating are covered in a dedicated video, check it out if you are interested. The inflation rate is adjusted by an equation that is dependent on the ideal staking percentage and currently staked DOTs. This model is subject to change when the parachains are live on the network. If you are a nominator deciding to choose Validators who can be trusted with your stake, then this targets tab can be very resourceful if you. By default, the list is sorted in descending order with respect to the expected returns on your nominated DOTs. You can also view the details of nominations under the waiting tab. If any slashes happened during this era, they would show up here. Finally, the individual validator stats can be viewed here.

Alright. Here is where you can explore the important events that are scheduled on the network. For instance, the start of new staking session event, election of new council candidates, start of new referendum voting period etc. Hope you got an intuition of what can be explored on the network using Polkadot JS Apps. If you would like to explore other networks on the polkadot ecosystem, you can switch to the respective network and start exploring!
